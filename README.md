---

## 🧠 Por que Código Limpo?

Grande parte do trabalho de um desenvolvedor não é escrever código novo, mas **ler, entender e modificar código existente**.

Segundo a literatura, **a maior parte do tempo em software é manutenção**, e código mal escrito gera:

- Dificuldade de entendimento
- Aumento de bugs
- Baixa produtividade
- Alto custo de manutenção

Código limpo não é apenas estética — é **profissionalismo**.

## 🔁 O que é Refatoração?

Refatoração é o processo de:

> **melhorar a estrutura interna do código sem alterar seu comportamento externo**

Ou seja:
- não muda o que o sistema faz
- melhora como o código está escrito

Refatorar é uma atividade contínua — não um evento isolado.

## 📏 Princípios Fundamentais

Ao longo deste material, você verá conceitos como:

- Nomes claros e expressivos
- Funções pequenas e com responsabilidade única
- Código legível (mais importante que “inteligente”)
- Redução de duplicação
- Baixo acoplamento e alta coesão
- Tratamento adequado de erros
- Testes como suporte à evolução do código

---

## 🧪 Mentalidade do Desenvolvedor

Código limpo não é apenas técnica — é atitude:

- Cuidar do código como um profissional
- Melhorar continuamente (Regra do Escoteiro)
- Evitar “deixar para depois”
- Ser responsável pela qualidade do que entrega
